# sprint_5
# Автотесты веб-приложения QA Desk

Проект содержит набор UI-автотестов для проверки основных пользовательских сценариев веб-приложения QA Desk. Тесты написаны на языке Python с использованием фреймворка pytest и библиотеки Selenium WebDriver.

Проверяются ключевые функции приложения: регистрация пользователей, авторизация, выход из системы и создание объявлений.

## Структура проекта

sprint_5/

  tests/  
    test_registration.py — тесты регистрации пользователя  
    test_login.py — тесты авторизации  
    test_logout.py — тесты выхода из системы  
    test_creation_ad.py — тесты создания объявлений  

  utils/  
    constants.py — тестовые данные и тексты сообщений  
    generator.py — генератор email и паролей  
    user.py — данные заранее созданного пользователя  
    waiters.py — константы ожиданий  

  locators/  
    locators.py — локаторы элементов интерфейса  

  conftest.py — общие фикстуры pytest  
  README.md

## Описание реализованных тестов

Регистрация пользователя (TestRegistration)

test_registration_valid_user — проверка успешной регистрации нового пользователя. Выполняется переход на страницу регистрации, заполнение формы валидными данными, создание аккаунта и проверка перехода на главную страницу. В правом верхнем углу отображаются аватар и имя пользователя.

test_registration_invalid_email — проверка регистрации с email, не соответствующим маске *******@*******.***. После отправки формы проверяется отображение сообщения «Ошибка» и подсветка полей Email, Пароль и Повторите пароль как ошибочных.

test_registration_existing_user — проверка регистрации уже существующего пользователя. При попытке создания аккаунта отображается сообщение об ошибке и подсвечиваются все обязательные поля формы.

Авторизация пользователя (TestLogin)

test_login_valid_user — проверка успешного входа в систему с валидными учетными данными. После авторизации осуществляется переход на главную страницу, где отображаются аватар и имя пользователя.

Выход из системы (TestLogout)

test_logout_user — проверка выхода авторизованного пользователя из системы. После нажатия кнопки «Выйти» аватар и имя пользователя перестают отображаться, а в правом верхнем углу появляется кнопка «Вход и регистрация».

Создание объявлений (TestCreationAd)

test_creation_ad_unauthorized_user — проверка попытки создания объявления неавторизованным пользователем. При нажатии кнопки «Разместить объявление» отображается сообщение «Чтобы разместить объявление, авторизуйтесь».

test_creation_ad_authorized_user — проверка создания объявления авторизованным пользователем. Выполняется вход в систему, заполнение всех обязательных полей формы (название, описание, стоимость), выбор категории и города из выпадающих списков, выбор состояния товара и публикация объявления. После этого осуществляется переход в профиль пользователя и проверяется наличие созданного объявления в разделе «Мои объявления».

## Фикстуры

driver — инициализация и закрытие браузера перед и после выполнения теста.  
login_user — авторизация пользователя перед выполнением теста.  
Генераторы тестовых данных используются для создания валидных и невалидных email и паролей.

## Архитектурные решения

Для повышения стабильности тестов используются явные ожидания WebDriverWait. Локаторы элементов интерфейса вынесены в отдельный модуль. Тестовая логика, тестовые данные и вспомогательные утилиты разделены по смысловым модулям.

## Запуск тестов

pytest
